Day 1 Pre-Homework - CSS Grid

Laying out our webpage is the first thing we do. Its like framing a house before deciding which wallpaper we'll put in. The skeleton of the page is undoubtedly the first part you should arrange everytime you work on a project. Therefore, it is only fitting that we introduce sooner rather than later a great way to build that frame.

CSS Grid is a property in CSS we can assign to selected HTML elements to lay out our page in a specific way. This is really useful to lay out your page quickly without messing with the box-model properties (we'll cover box-model later on). In tandem with media-queries, we can use CSS grid to quickly build layouts for desktop, tablet or phone!

The parts of CSS Grid

There are many new terms to learn in CSS Grid, but let's just start with the most essential:

To set an area of your webpage, say, your <body/> simply selecte the body in your CSS file and set the property display to grid like so: 

body {
  display: grid;
}

This body is now called our "Grid Container". You will need one of these parent grid containers everytime you would like to arrange elements, or "grid items", inside it.

After that you'll need to set your container to have the number of columns and rows you desire. Let's say we want our body to have 2 equally sized columns and 4 equally sized rows: 

body {
  display: grid;
  grid-template-columns: 50% 50%;
  grid-template-rows: 25% 25% 25% 25%;
}

Now our grid has two columns that are 50% of the display and 4 rows that are each 25% or the display. We simply type in percentages, pixels or ems as many times as we want rows. And its the same for rows.

Then we can place our grid items inside the grid container. Let say we want to our HTML element with the class name: our-grid-item to be positioned at the bottom-right corner of our grid (in the last row of the right column)....

.our-grid-item {
  grid-column: 2/3;
  grid-row: 4/5;
}

Do you see that? We refer to the grid-lines to tell the browser where to place the element! Use the property grid-column and give it the value you want the element to start and end on separated by a "/". This tells the browser to find the line in the grid you said specified on the left side of the "/" and end on the number you gave on the right side. Here it is again but now our element is in the top-left of our document (top-row of left-column).

.our-grid-item {
  grid-column: 1/2;
  grid-row: 1/2;
}

Deeper Dive into CSS Grid

Before looking at the code below, read this article. Chen you finish, come back and see if you can follow along.

  • To start out let's layout our elements in HTML, and give each grid-area a class name.
<div class="container">
  <header>header</header>
  <nav>
    <ul>
      <li><a href="#">nav 1</a></li>
      <li><a href="#">nav 2</a></li>
      <li><a href="#">nav 3</a></li>
    </ul>
  </nav>
  <main>
    <h1>Main Title</h1>
    <p>Main content</p>
  </main>
  <aside>aside</aside>
  <div class="banner">div.banner</div>
  <footer>footer</footer>
</div>
  • In our CSS file we declare each HTML element as a grid-area and pass it the name we want to use when we specify where they go in our grid-template-areas
header {
  grid-area: header;
}
main {
  grid-area: main;
}
nav {
  grid-area: nav;
}
aside {
  grid-area: aside;
}
.banner {
  grid-area: banner;
}
footer {
  grid-area: footer;
}
  • Then we define how it will look on the device. Remember, we build mobile first. Notice there are no media queries.
/* ... */

.container {
  display: grid; /* Using the grid */
  grid-template-areas: /* Notice how I've stacked my areas, just like they look on the page! */
    "header"
    "nav"
    "main"
    "aside"
    "banner"
    "footer";
}

  • Now if our screen gets a little bigger (say 500px), we want our layout to stack more horizontally
/* ... */

@media (min-width: 500px) {
  .container {
    display: grid;
    grid-template-columns: 1fr 3fr; /* the first column is 1 unit (fraction), the second column is 3 units */
    grid-template-areas: /* stack the components how you want them to appear in the columns */
      "header  header" /* stretch header across both columns at the top */
      "nav     nav"
      "aside   main" /* stack these horizontally, aside first, etc */
      "banner  footer";
  }
  nav ul {
    display: flex;
    justify-content: space-between;
  }
}

And then we can have something different happen when we get on a larger screen (700px+) using our code?

/* ... */

@media (min-width: 700px) {
  .container {
    grid-template-columns: 1fr 4fr 1fr;
    grid-template-areas: 
      "header header  header"
      "nav    main    aside"
      "nav    main    banner"
      "footer footer  footer"
   }
   nav ul {
     flex-direction: column;
   }
}

**There are a few other properties you can use with css-grid. Take a look at these.** 1. `grid-column-gap: 1em;` will set the width between columns to 1[em](https://www.w3schools.com/cssref/css_units.asp). 1. `grid-row-gap: 1em;` will set the width between rows. Try it out below with the following example:
**More tips and tricks:** 1. Instead of writing multiple `fr`'s in `grid-template-columns: 1fr 1fr 1fr;`, you can use `grid-template-columns:repeat(3, 1fr);`. 1. To set fixed row heights use `grid-auto-rows: 100px;`. This makes all row heights 100px. 1. To set flexible row heights use `grid-auto-rows: minmax(100px, auto);`. This gives all row heights a minimum size of 100px and a maximum hieght that will fit all their content. Play with these additions below.
